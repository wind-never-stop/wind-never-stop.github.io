四、shiro+jwt+redis+rbac实现菜单+按钮权限控制

访问链接：http://60.205.177.220:8081/#/roleauth
服务端调用说明：https://maqi.coding.net/p/dianxue-admin/wiki/4

    1. rbac（role base access controll）基于角色的权限控制，5张表
  • 2.jwt（java web token） 基于令牌的身份识别
    • 登陆信息验证通过：计算生成带有效期的token令牌，客户端保存
    • 发起资源请求时携带token令牌，服务在filter中进行过滤验证
  • 3.shiro 在通过token识别出用户身份基础上，做细粒度权限控制
    • 登陆信息验证通过：将用户拥有的权限返回给前端，控制菜单按钮的展示
    • 发起资源请求时通过携带token令牌，识别出用户身份，校验用户是否对该url有访问权限
  • 4.redis
    • 缓存用户权限信息
    • 做单点登陆控制

七、easy-code template/ generator 自动生成基本功能的增删改查：

  • 1.idea plugin easy-code生成代码

    • 参考：https://gitee.com/makejava/EasyCode/wikis/pages
    • 原理：使用Velocity模版生成代码，可自定义修改代码生成模版
    • 使用
      • 配置idea database 数据源
      • 配置table column 映射java bean规则
      • 编写修改调试Easy Template:Velocity模板
      • 选择要生成的模板组，package、file路径，点击OK生成

  • 2.generator

    • 原理：采用mapper.java 模板类的形式，读取db table info，对模板类中的占位符进行replace，逐个替换内容生成
    • 使用时，需要配置db，模板类、位置java类输出文件位置、package路径、java业务类后缀
  • 备注：目前还有很多地方不完善
