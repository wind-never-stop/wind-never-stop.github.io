四个方向的自己管理

  • 向左（客户）要有爱（放下自己）
  • 向右（协同部门、同事）要有肺（放下”私心”）
  • 向上（上级）要有胆（放下小心、协助上级打胜仗，事先想过认为正确的问题可以提出，和顶撞是有区分的）

做成熟男人

发表于 2021-11-19 分类于 人生

前言

做个男人，做个成熟的男人，做个有城府的男人

突然间觉得，自己已经不是青涩时候的自己了。应该正视家人的叽叽喳喳，领导的唧唧歪歪，同事的稀稀拉拉，同学朋友们的嘻嘻哈哈。
看了一篇帖子，讲男人的，觉得说的很好，拿过来，鞭策自己，同时鞭策能看到这篇文章的朋友们，男人们。
是谁说过的，一个人的成熟程度，与其的情商是成正比的，一个人成事的概率也与其情商是成正比的。

一：沉稳

（1）不要随便显露你的情绪。
（2）不要逢人就诉说你的困难和遭遇。
（3）在征询别人的意见之前，自己先思考，但不要先讲。
（4）不要一有机会就唠叨你的不满。
（5）重要的决定尽量有别人商量，最好隔一天再发布。
（6）讲话不要有任何的慌张，走路也是。
（7）自信是好，但是别忽略任何人的想法。
（8）人无高低，不要一副拽拽的样子，对人对事，别忘了礼貌。你没有比任何人优秀。

活动抽奖榜单实现

发表于 2021-11-19 分类于 java ， redis

背景

  • 提供给用户的高性能接口
  • 返回活动当天抽中礼物价值总和靠前的 用户排行榜
  • 返回活动当天自己每日抽中的礼物价值总和

流程

  • 查询redis key=activityId+day 存在返回
  • 不存在
    • 查询es 排名 + 50预留排名，接口异常、数据异常则抛错结束
    • 正常返回
      • 异步写入redis，key=activityId+day
      • 返回数据
  • 查询上榜用户冻结状态，接口异常兜底为非冻结
  • 过滤冻结用户后返回榜单前N
  • 查询活动自己每日抽中的礼物价值总和

不足

请求量过大，缓存失效间隙场景会出现耗时尖刺，部分用户体验较差

优化

提供定时主动刷新缓存机制

使用定时任务获取es数据，定时刷新redis值，保证用户查询时，缓存中的值均为最新榜单，消除耗时尖刺
