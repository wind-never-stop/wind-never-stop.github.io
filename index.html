订单业务及架构介绍

发表于 2022-05-04 分类于 java

未完待续。

1.业务背景

从毕业以来，一直在从事订单交易方向的后端研发，历经两家公司，交易体量从小到大，见证了订单架构的演进变化，踩过不少坑，特此反思总结下来，希望未来的路能越走越远。

1.1电商交易订单

1.1.1核心流程
  1. 用户从PC、小程序、APP、门店收银机等终端设备浏览商品；
  2. 加入购物车或者直接购买下单，进入订单购买页，选择优惠券、积分、兑换券等提交订单，进入订单结算页面；
  3. 通过微信、支付宝、银联等第三方服务商完成支付，订单推送到商家订单侧，进入待发货环节；
  4. 商家通过实体物流发货，填写上传物流单号，订单流转到已发货；
  5. 用户在平台中可查看实时物流信息，到货验货，完成订单。
1.1.2系统指标

承接、预售、拼团、超级团、营销三宝、闪购、众筹等玩法

2017-2019一条项目总结

发表于 2022-03-07

订单系统-下单流程-技术细节
dubbo配置限流 executes –》 2.6.x ExecuteLimitFilter->Semaphare 信号量

业务代码降级：
    1）同步调用重试
    2）核心流程返回正常，异步发消息重试、db存储一个死信队列，用quartz重发到消息队列中 

分布式锁做防重复提交订单：
    1）渲染订单下发submitKey，提交订单传递submitKey --> 表达页面过期
    2）redis: set("nx", "ex", expireTime)->"OK" --> 获取分布式锁成功
    3）zookeeper: 临时目录节点 --> 创建节点
        判断刚创建的节点是否为第一顺序节点 
        --> true：加锁成功
        --> false: 注册一个回调监听前一个节点的释放动作
            （回调内容，判断自己创建的节点是否为第一顺序节点-> true:加锁成功）

分布式事务实现最终一致性：tcc:try->confirm->cancel 
    1) try方法上提供注解：指定事务完成（confirmMethod）或回滚需要调用的方法(cancelMethod)
    2) AOP实现：
        a) 调用try方法后，调用tcc.uploadStatus dubbo接口（状态保存到redis list：lpush lrange lset del scan，根据try是否抛异常来异步流转事务状态）
        b) 根据主调方 try方法是否抛异常来决定 tcc call confirmMethod or cancelMethod，注意这两方法会要求传入 transId, branchId
    3）举例：以库存接入tcc事务为例
        a) 声明并实现 try，注意服务按需自行保存 事务提交 or 回归的必要参数，也就是 transactionLog（一般被调方需要保存）
        b) 声明并实现 confirm 和 cancel 方法，根据传入的 transId, branchId 从 transactionLog 获取必要参数进行后续逻辑
        c) spring配置文件声明 AspectBean 启用 tcc aop

高并发扣减库存：库存存放于redis，如何保证最终一致性:记param-log
    1）try阶段：pipeline.multi()-->submit()
        save reduceStockParamsLog to redis 
        reduce redis skuStock
    2）confirm阶段：delete reduceStockParamsLog
    3）cancel阶段：pipeline.multi()-->submit()
        return redis skuStock
        delete reduceStockParamsLog

四个方向的自己管理

  • 向左（客户）要有爱（放下自己）
  • 向右（协同部门、同事）要有肺（放下”私心”）
  • 向上（上级）要有胆（放下小心、协助上级打胜仗，事先想过认为正确的问题可以提出，和顶撞是有区分的）
  • 向下（下级）要有心（放下架子，做良师益友）

做成熟男人

发表于 2021-11-19 分类于 人生

前言

做个男人，做个成熟的男人，做个有城府的男人

突然间觉得，自己已经不是青涩时候的自己了。应该正视家人的叽叽喳喳，领导的唧唧歪歪，同事的稀稀拉拉，同学朋友们的嘻嘻哈哈。
看了一篇帖子，讲男人的，觉得说的很好，拿过来，鞭策自己，同时鞭策能看到这篇文章的朋友们，男人们。
是谁说过的，一个人的成熟程度，与其的情商是成正比的，一个人成事的概率也与其情商是成正比的。

一：沉稳

（1）不要随便显露你的情绪。
（2）不要逢人就诉说你的困难和遭遇。
（3）在征询别人的意见之前，自己先思考，但不要先讲。
（4）不要一有机会就唠叨你的不满。
（5）重要的决定尽量有别人商量，最好隔一天再发布。
（6）讲话不要有任何的慌张，走路也是。
（7）自信是好，但是别忽略任何人的想法。
（8）人无高低，不要一副拽拽的样子，对人对事，别忘了礼貌。你没有比任何人优秀。
